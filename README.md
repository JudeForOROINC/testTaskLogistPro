Техническое задание:
1) Предметная область задачи (не связана с деятельностью компании)
2) Что необходимо учесть при выполнении задания (технические ограничения):
 Любой современный фреймворк;
 Любая БД;
 Общение между фреймворком и БД не должно основываться на
 ActiveRecord (т.е. должны использоваться другие виды ORM);
 Реализация в виде API, формат данных JSON; Есть Персона, у нее есть
 ФИО, ИНН и активы. У актива есть название, стоимость и тип (движимый или недвижимый).
 Для Персоны посчитать кредитный лимит.
 Стратегия расчета по умолчанию:
 1,1 * (сумма стоимостей недвижимых активов) + 0,9 * (сумма
 стоимостей движимых активов) Альтернативная стратегия расчета:
 0,8 * (сумма стоимостей недвижимых активов) + 1,5 * (сумма
 стоимостей движимых активов) API должен быть следующим:
 POST /api/person - создание Персоны (с активами) GET
 /api/person/{tin}
 - просмотр Персоны PATCH /api/person/{tin} - изменение Персоны
 DELETE /api/person/{tin} - удаление Персоны GET
 /api/person/{tin}/limit - расчет лимита для Персоны стратегией по
 умолчанию GET /api/person/{tin}/limit?strategy=alternate - расчет
 лимита для Персоны альтернативной стратегией Под результатом расчета
 подразумевается JSON-ответ в виде {
 result: "для {ФИО} кредитный лимит составляет {число}"
 }
 *tin - это ИНН
